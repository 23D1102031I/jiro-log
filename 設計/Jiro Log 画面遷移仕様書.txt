Jiro Log 画面遷移仕様書
1. グローバルナビゲーション（共通ヘッダー）
全画面の上部に固定で配置され、主要な5画面への遷移を司ります。

🏠 ホーム -> ホーム画面へ遷移

📍 店舗マップ -> 店舗マップ画面へ遷移

👑 スタンプカード -> スタンプカード画面へ遷移

✏️ レビュー投稿 -> レビュー投稿画面へ遷移

👤 マイページ -> マイページ画面へ遷移

2. 各画面の構成と遷移定義
① ホーム画面 (Home)
アプリのポータルとなる画面。各機能へのハブとしての役割を持ちます。

画面構成要素:

最新の神豚投稿（ヒーローバナー）

最新レビュータイムライン

直系コンプリートへの道（スタンプカード進捗プレビュー）

店舗マップ（プレビュー） / 二郎ルール（基本）

店舗ランキング / 最新の称号獲得者 / 人気の称号

フローティングアクションボタン（レビューを投稿する）

遷移先 (アクション):

タイムラインの各レビュータップ -> レビュー詳細画面（※想定）

「すべて見る」タップ -> タイムライン一覧画面（※想定）

「スタンプカードを見る」ボタン -> スタンプカード画面

「店舗マップを開く」ボタン -> 店舗マップ画面

「称号一覧を見る」ボタン -> 称号一覧・マスタ画面（※想定）

「レビューを投稿する」ボタン -> レビュー投稿画面

② 店舗マップ画面 (Store Map)
現在地周辺や特定のエリアから直系店舗を探す画面です。

画面構成要素:

検索バー / 絞り込みフィルター（エリア、営業状況等）

店舗リスト（左側カラム）

インタラクティブマップ（右側エリア）

選択中店舗の詳細スニペット（マップ下部）

遷移先 (アクション):

マップ上のピン、または左側のリストをタップ -> 店舗のプレビュー情報が下部に表示される（状態変化）

プレビュー情報の「詳細ページへ」ボタン -> 店舗詳細画面（※想定）

③ スタンプカード画面 (Stamp Card)
直系全店制覇のモチベーションを高める実績画面です。

画面構成要素:

全44店舗のスタンプグリッド（「完食」と「未訪問」でステータス表示）

現在の制覇状況（ドーナツチャート、◯/44店）

獲得した最高位の称号（例：ロットの支配者）

遷移先 (アクション):

各店舗のスタンプ（アイコン）をタップ -> 該当の店舗詳細画面（※想定）

④ レビュー投稿画面 (Review Post)
食べた一杯の詳細を記録する入力フォーム画面です。

画面構成要素:

ラーメン画像アップローダー（ドラッグ＆ドロップ対応）

コール（呪文）選択ボタン（ニンニク、ヤサイ、アブラ、カラメ）

総合評価（★1〜5）

コメント入力エリア

ジロリアンパラメータ（6項目の独自スライダー）

遷移先 (アクション):

「この一杯を記録する」ボタン -> 投稿完了画面（※想定） または マイページ（レビュー履歴） に遷移し、トースト通知を表示。

⑤ マイページ画面 (My Page)
ユーザー自身のプロフィールと、二郎の嗜好・実績を振り返る画面です。

画面構成要素:

ユーザープロフィール（アイコン、ユーザー名、最高位称号）

サマリー数値（総実食数、制覇店舗、レビュー数、総いいね数）

My Jiro Identity（6項目の平均値レーダーチャート＆スライダー）

獲得した称号一覧

レビュー履歴リスト

遷移先 (アクション):

「称号一覧を見る」または「もっと見る」ボタン -> 自身の獲得称号一覧画面（※想定）

レビュー履歴の店舗名・絞り込み -> 該当店舗のレビュー一覧（※想定）

レビュー履歴の各項目タップ -> 対象のレビュー詳細画面（※想定）

💡 補足：実装に向けたルーティング設計（Next.js App Router 想定）
要件定義書に Next.js (App Router) とあるため、ディレクトリ構成（URLパス）は以下のように設計するとスムーズです。

/ (ホーム)

/map (店舗マップ)

/stamp (スタンプカード)

/post (レビュー投稿)

/mypage (マイページ)

/stores/[id] (店舗詳細 - ※画面遷移先として想定)

/reviews/[id] (レビュー詳細 - ※画面遷移先として想定)
